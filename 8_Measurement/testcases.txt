--cpu: p                                                -> +, +p
--io: l / s                                             -> +
--vm: bof
--hdd: l
--sem: bof
--switch: bl                                            -> +
--timer: bof
--poll: bof
--fork: ls
--fallocate: ls -> good for s because disk sleep        -> +, +s
--dentry: bls                                           -> +, +b
--cache: l
--affinity: l

p  prio
l  lock
b  block
s  sleep
c  comp
sc sched

--cpu:       start N processes computing sqrt((double)rand())
--io:        start N processes continuously calling sync() to commit buffer cache to disk
--switch:    start N processes that send messages via pipe to a child to force context switching
--fallocate: start N processes continally fallocating (preallocating file space) and ftuncating (file truncating) temporary files.
--dentry:    start N processes that create and remove directory entries

synch: sync writes any data buffered in memory out to disk
fallocate: allows the caller to directly manipulate the allocated disk space for the file
directory entry: Directory Entry is basically the mapping of filename to its inode
                 inode contains info about a file (type, permission, size, )
